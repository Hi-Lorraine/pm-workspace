失活客户智能召回 — MCP 接口需求评审

产品负责人：淼鑫  |  日期：2026-04-03  |  场景：数字BD分身 · 失活客户召回

评审目标：确认失活召回场景所需的 MCP 工具接口定义（输入参数 + 输出字段），研发完成 MCP 封装后，产品侧基于此编写 Skills。

一、业务流程概览

定时扫描（系统每日自动）→ 规则触发（识别失活客户）→ 风险评估（AI 计算流失等级）→ 话术生成（AI 个性化召回消息）→ 消息触达（企微/语音）→ 结果推送（通知责任BD跟进）

失活等级定义

失活等级规则触发动作
🟡 轻度失活连续 7 天未登录企微自动发送关怀消息
🟠 中度失活2 个月未登录AI 生成个性化召回话术 + 企微触达
🔴 重度失活3 个月未下单AI 语音外呼 + 推送 BD 人工跟进

二、Skill → MCP 调用关系

Skill功能调用的 MCP 工具
失活风险扫描按规则筛选失活客户，输出风险列表工具1 + 工具2
个性化话术生成根据客户画像生成定制化召回消息工具2 + 工具3
消息触达通过企微/语音触达客户，结果推送给BD工具4 + 工具5

工具 1：查询 BD 名下失活客户列表

核心接口。BD 问"我有哪些客户快流失了"，或系统定时扫描触发。

输入参数

参数类型必填说明
bd_idstring必填BD 工号，从登录态自动注入
inactive_typestring选填失活等级：light(7天未登录) / medium(2月未登录) / severe(3月未下单)，不传返回全部
no_login_daysint选填自定义未登录天数阈值（灵活查询，覆盖 inactive_type）
no_order_daysint选填自定义未下单天数阈值
customer_levelstring选填按客户等级过滤：A / B / C / D
sort_bystring选填排序：risk_level / last_login / purchase_amount，默认 risk_level
pageint选填分页页码，默认 1
page_sizeint选填每页条数，默认 20

输出字段

字段类型说明
customer_idstring客户ID
customer_namestring药店名称
customer_levelstring客户等级（A/B/C/D）
customer_typestring连锁 / 单体 / 诊所
inactive_typestring失活等级：light / medium / severe
last_login_datedate最近登录日期
last_order_datedate最近下单日期
days_since_loginint距上次登录天数
days_since_orderint距上次下单天数
purchase_amount_90dfloat近90天采购额
purchase_amount_prev_90dfloat前90天采购额（环比用）
contact_phonestring联系电话
total_countint符合条件的总客户数

AI 生成个性化话术前，必须先获取客户完整画像信息。

输入参数

参数类型必填说明
bd_idstring必填权限校验，只能查自己名下客户
customer_idstring必填客户ID

输出字段

字段类型说明
customer_namestring药店名称
customer_levelstring等级（A/B/C/D）
customer_typestring连锁 / 单体 / 诊所
regionstring所在区域
bindbd_namestring归属 BD 姓名
last_login_datedate最近登录日期
last_order_datedate最近下单日期
purchase_amount_30dfloat近30天采购额
purchase_amount_90dfloat近90天采购额
purchase_amount_prev_90dfloat前90天采购额
purchase_trendstring采购趋势：上升 / 持平 / 下降 / 断崖
order_count_90dint近90天订单数
avg_order_amountfloat平均客单价
top_categoriesarray常购品类 TOP3（品类名 + 金额占比）
top_productsarray常购商品 TOP5（商品名 + 近90天采购量）
first_order_datedate首单日期（客户生命周期）
contact_phonestring联系电话
last_contact_datedateBD 上次联系时间

工具 3：查询客户采购历史

生成话术时引用"您上次买的XX"做个性化推荐，提升召回效果。

输入参数

参数类型必填说明
bd_idstring必填权限校验
customer_idstring必填客户ID
date_fromdate选填开始日期，默认近90天
date_todate选填截止日期，默认今天
categorystring选填按品类过滤
page_sizeint选填默认 10

输出字段

字段类型说明
order_idstring订单号
order_datedate下单日期
total_amountfloat订单金额
item_countint商品种类数
top_itemsarray主要商品（商品名 + 数量 + 金额）
order_statusstring订单状态

工具 4：发送企微消息

触达客户。轻度失活自动发关怀、中度发召回话术。如企微通道不通，MVP 降级为推送给 BD 手动发。

输入参数

参数类型必填说明
bd_idstring必填发送人（BD 身份）
customer_idstring必填接收客户
message_typestring必填text（纯文字）/ card（客户卡片）/ voice_script（语音话术脚本）
contentstring必填消息内容（AI 生成的话术）
scheduled_timedatetime选填定时发送时间，不传则立即发送

输出字段

字段类型说明
send_statusstringsuccess / failed / pending
send_timedatetime实际发送时间
fail_reasonstring失败原因（如：客户非好友、消息频率限制等）

工具 5：推送提醒给 BD

召回结果或高风险预警推送给责任 BD，闭环最后一步。

输入参数

参数类型必填说明
bd_idstring必填目标 BD
notify_typestring必填risk_alert（流失预警）/ recall_result（召回结果）/ recall_failed（触达失败需人工跟进）
customer_idstring必填涉及的客户
titlestring必填提醒标题，如"重度失活预警：XX大药房"
contentstring必填提醒内容摘要
prioritystring选填high / medium / low，默认 medium

输出字段

字段类型说明
notify_statusstringsuccess / failed
notify_channelstring实际推送渠道（工作台 / 企微 / 短信）

权限规则（所有接口通用）

规则说明
BD 只能查自己名下客户所有接口必须校验 bd_id 与 customer_id 的归属关系
bd_id 从登录态注入前端不传 bd_id，由中间件从 token 解析注入，防伪造
越权访问拦截非名下客户返回"该客户不在您名下"提示，不返回数据
敏感字段脱敏手机号中间四位脱敏（如 138****5678），完整号码需二次确认

需研发确认的数据问题

以下问题直接决定接口能否实现、字段是否可用，请评审会上逐条确认。
#问题影响范围如果没有的降级方案
1客户登录日志有吗？能拿到 last_login_date？7天/2月未登录规则、工具1和工具2退化为仅用"未下单天数"判断失活
2BD 与客户归属关系在哪个系统/表？所有接口的权限基础无降级，必须有
3客户等级数据在哪？实时还是离线标签？筛选过滤 + 话术个性化先按采购额粗分档
4历史采购能按时间范围聚合吗？还是只有原始订单？环比趋势计算研发需先做聚合再封装
5常购品类/商品有现成标签吗？工具2的 top_categories / top_products从订单明细实时统计（性能注意）
6企微消息通道现在通不通？BD和客户是否已是好友？工具4能否实现MVP 降级：AI 生成话术 → 推给BD手动发
7这些数据现在有 API 还是只有数据库/数仓？研发工作量（有API包一层，没有要先写API）—
8BD 上次联系客户的记录在哪？CRM 有通话/拜访记录吗？工具2的 last_contact_dateMVP 先不出这个字段
