失活客户智能召回 — MCP 接口需求评审

产品负责人：淼鑫  |  日期：2026-04-03  |  场景：数字BD分身 · 失活客户召回

评审目标：确认失活召回场景所需的 8 个 MCP 工具接口定义（输入 + 输出），研发完成 MCP 封装后，产品侧基于此编写 4 个 Skills，实现从识别 → 触达 → 追踪的完整闭环。

一、业务流程 & 闭环

每日定时扫描（工具1）→ 识别失活客户 → 按等级分流：
  🟡 轻度 → AI 生成关怀话术（工具2+3）→ 查企微好友（工具6）→ 自动发送（工具4）→ 记录（工具7）
  🟠 中度 → AI 生成召回话术（工具2+3）→ BD 确认 → 查好友（工具6）→ 触达（工具4）→ 记录（工具7）
  🔴 重度 → 推送给 BD 人工跟进（工具5）→ 记录（工具7）
        ↓ 7天后 ↓
  追踪召回效果（工具8）→ 回流 ✅ / 未响应 → 升级处理

失活等级定义

等级规则触发动作
🟡 轻度连续 7 天未登录企微自动发送关怀消息
🟠 中度2 个月未登录AI 个性化召回话术 + 企微触达
🔴 重度3 个月未下单推送 BD 人工跟进（二期加 AI 语音外呼）

二、Skill → MCP 工具映射

Skill功能调用工具
Skill 1 失活风险扫描按规则筛选失活客户，输出风险列表工具1 + 工具2
Skill 2 个性化话术生成根据客户画像 + 采购历史生成召回话术工具2 + 工具3
Skill 3 一键触达校验好友关系 → 企微发送 / 降级推BD → 写入记录工具6 → 工具4 / 工具5 → 工具7
Skill 4 召回效果复盘追踪客户是否回流，汇总召回成功率工具8

三、MCP 工具定义（共 8 个）

工具 1：查询 BD 名下失活客户列表

BD 问"我有哪些客户快流失了" / 系统每日定时扫描

输入参数类型说明
bd_idstring必填从登录态注入
inactive_typestring选填light / medium / severe，不传返回全部
no_login_daysint选填自定义未登录天数（覆盖 inactive_type）
no_order_daysint选填自定义未下单天数
customer_levelstring选填按客户等级过滤 A/B/C/D
exclude_contacted_daysint选填排除 N 天内已触达过的客户，默认 7
sort_bystring选填risk_level / last_login / purchase_amount
page / page_sizeint选填分页，默认 1 / 20
输出字段类型说明
customer_idstring客户ID
customer_namestring药店名称
customer_levelstring等级 A/B/C/D
customer_typestring连锁 / 单体 / 诊所
inactive_typestringlight / medium / severe
last_login_datedate最近登录日期
last_order_datedate最近下单日期
days_since_loginint距上次登录天数
days_since_orderint距上次下单天数
purchase_amount_90dfloat近90天采购额
purchase_amount_prev_90dfloat前90天采购额（环比用）
last_recall_datedate上次召回触达时间（null=从未触达）
last_recall_resultstring上次触达结果：已回复/未回复/已回流/无效
contact_phonestring联系电话（脱敏）
total_countint符合条件的总客户数

工具 2：查询单个客户详细画像

AI 生成话术前获取客户背景；BD 问"这个客户什么情况"

输入类型说明
bd_idstring必填权限校验
customer_idstring必填客户ID
输出字段类型说明
customer_namestring药店名称
customer_levelstring等级
customer_typestring连锁 / 单体 / 诊所
regionstring所在区域
bindbd_namestring归属 BD
last_login_datedate最近登录
last_order_datedate最近下单
purchase_amount_30d / 90d / prev_90dfloat近30天 / 近90天 / 前90天采购额
purchase_trendstring上升 / 持平 / 下降 / 断崖
order_count_90dint近90天订单数
avg_order_amountfloat平均客单价
top_categoriesarray常购品类 TOP3
top_productsarray常购商品 TOP5
first_order_datedate首单日期
contact_phonestring联系电话（脱敏）
last_contact_datedateBD 上次联系时间

工具 3：查询客户采购历史

生成话术时引用"您上次买的XX"做个性化推荐

输入类型说明
bd_idstring必填权限校验
customer_idstring必填客户ID
date_from / date_todate选填默认近90天
categorystring选填按品类过滤
page_sizeint选填默认 10
输出字段类型说明
order_idstring订单号
order_datedate下单日期
total_amountfloat订单金额
item_countint商品种类数
top_itemsarray主要商品（名称 + 数量 + 金额）
order_statusstring状态

工具 4：发送企微消息

触达客户。企微不通则降级为推送给 BD 手动发

输入类型说明
bd_idstring必填发送人
customer_idstring必填接收客户
external_useridstring必填企微外部联系人ID（从工具6获取）
message_typestring必填text / card / voice_script
contentstring必填AI 生成的话术内容
scheduled_timedatetime选填定时发送，不传立即发送
输出字段类型说明
send_statusstringsuccess / failed / pending
send_timedatetime实际发送时间
fail_reasonstring失败原因（非好友 / 频率限制等）

工具 5：推送提醒给 BD

流失预警 / 触达失败 / 召回结果推送给责任 BD

输入类型说明
bd_idstring必填目标 BD
notify_typestring必填risk_alert / recall_result / recall_failed
customer_idstring必填涉及的客户
titlestring必填提醒标题
contentstring必填内容摘要
prioritystring选填high / medium / low
输出类型说明
notify_statusstringsuccess / failed
notify_channelstring工作台 / 企微 / 短信

工具 6：查询企微好友关系

发消息前置校验——不是好友发不了，需降级为推送给 BD 手动处理

输入类型说明
bd_idstring必填BD 工号
customer_idsarray必填客户ID列表（支持批量）
输出类型说明
customer_idstring客户ID
is_friendbool是否为企微好友
external_useridstring企微外部联系人ID（发消息用）
friend_sincedate加好友时间

工具 7：记录召回动作

每次触达写一条记录，用于去重、追踪、统计

输入类型说明
bd_idstring必填操作人
customer_idstring必填触达客户
recall_typestring必填auto_qywx / auto_voice / manual_bd
inactive_typestring必填light / medium / severe
message_contentstring必填发送的话术内容
输出类型说明
recall_idstring召回记录ID
statusstringcreated

工具 8：查询召回效果

闭环追踪：客户触达后是否回流、是否下单

输入类型说明
bd_idstring必填BD
date_from / date_todate选填召回日期范围
recall_resultstring选填returned / no_response / all
输出类型说明
recall_idstring召回记录ID
customer_id / customer_namestring客户
recall_datedate触达日期
recall_typestring触达方式
inactive_typestring当时失活等级
has_login_afterbool触达后是否有登录
has_order_afterbool触达后是否有下单
first_order_afterdate触达后首次下单时间
order_amount_after_7dfloat触达后7天内采购额
recall_resultstringreturned / no_response / lost

四、权限规则（所有接口通用）

规则说明
BD 只查自己名下客户所有接口校验 bd_id 与 customer_id 归属关系
bd_id 从登录态注入中间件从 token 解析，前端不传，防伪造
越权拦截非名下客户返回"该客户不在您名下"，不返回数据
手机号脱敏中间四位脱敏（138****5678），完整号码需二次确认
